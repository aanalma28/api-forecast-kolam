API Documentation

The ML Forecasting API provides powerful machine learning-based time series forecasting capabilities through a simple REST interface.

This API is designed for developers who need to integrate forecasting functionality into their applications, offering endpoints for single and batch predictions with comprehensive error handling and validation.

Base URL: http://localhost:5000/api

Authentication

Currently, the API does not require authentication. All endpoints are publicly accessible.

Note: In production environments, proper authentication should be implemented.

API Endpoints

GET Health Check

Endpoint: /api/health

Check the health status of the API and model.

Response:
{
  "status": "healthy",
  "timestamp": "2024-01-01T12:00:00.000Z",
  "model_status": "trained",
  "version": "1.0.0"
}

GET Model Information

Endpoint: /api/model/info

Get information about the loaded forecasting model.

Response:
{
  "success": true,
  "data": {
    "model_type": "Linear Regression with Time Features",
    "is_trained": true,
    "features": ["trend", "sin_seasonal", "cos_seasonal", "month", "day_of_week"],
    "model_path": "data/sample_model.pkl"
  },
  "timestamp": "2024-01-01T12:00:00.000Z"
}

POST Generate Forecast

Endpoint: /api/forecast

Generate forecasting predictions for a specified time period.

Request Body:
{
  "start_date": "2024-01-01",  // Optional, defaults to today
  "periods": 30,               // Optional, defaults to 30 (max 365)
  "confidence_level": 0.95     // Optional, defaults to 0.95
}
Response:
{
  "success": true,
  "data": {
    "predictions": [
      {
        "date": "2024-01-01",
        "predicted_value": 125.67,
        "period": 1
      }
      // ... more predictions
    ],
    "summary": {
      "total_periods": 30,
      "min_value": 120.45,
      "max_value": 135.89,
      "mean_value": 128.12,
      "start_date": "2024-01-01",
      "end_date": "2024-01-30"
    },
    "metadata": {
      "model_type": "Linear Regression Forecast",
      "generated_at": "2024-01-01T12:00:00.000Z",
      "parameters": {
        "start_date": "2024-01-01",
        "periods": 30,
        "confidence_level": 0.95
      }
    }
  }
}

POST Batch Forecast

Endpoint: /api/forecast/batch

Generate multiple forecasts in a single request (max 10 requests per batch).

Request Body:
{
  "requests": [
    {
      "start_date": "2024-01-01",
      "periods": 15
    },
    {
      "start_date": "2024-02-01",
      "periods": 30
    }
  ]
}
Response:
{
  "success": true,
  "data": {
    "results": [
      {
        "request_id": 0,
        "success": true,
        "data": [/* predictions array */]
      },
      {
        "request_id": 1,
        "success": true,
        "data": [/* predictions array */]
      }
    ],
    "total_requests": 2,
    "successful_requests": 2,
    "generated_at": "2024-01-01T12:00:00.000Z"
  }
}

Usage Examples

JavaScript (Fetch)
const response = await fetch('/api/forecast', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    start_date: '2024-01-01',
    periods: 30
  })
});

const data = await response.json();
console.log(data.data.predictions);
Python (requests)
import requests

response = requests.post('/api/forecast', json={
    'start_date': '2024-01-01',
    'periods': 30
})

data = response.json()
predictions = data['data']['predictions']

Error Handling

The API returns standardized error responses for various failure scenarios:

Validation Error (400):
{
  "success": false,
  "error": "Validation failed",
  "details": ["periods must be at least 1"],
  "status": 400
}
Rate Limit Error (429):
{
  "error": "Rate limit exceeded",
  "message": "Maximum 100 requests per hour allowed",
  "status": 429
}
Server Error (500):
{
  "success": false,
  "error": "Internal server error",
  "message": "Failed to generate forecast predictions",
  "status": 500
}

Rate Limiting

The API implements rate limiting to ensure fair usage:

  • Limit: 100 requests per hour per IP address
  • Window: Rolling 1-hour window
  • Response: HTTP 429 when limit exceeded
Rate limits are applied per client IP address and reset every hour.
